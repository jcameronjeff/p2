## Usage

<definition>
  Accordions provide a quick way to vertically show or hide content and
  functionality.
</definition>

### Common Characteristics

All accordions have a label that toggles content below it to show or hide.

A 150ms transition is used to expand the content area down, and 75ms to collapse it up.

### Filter Accordion

The filter accordion accommodates a clear function that clears all selections within a filter category. It is hidden when there are no selections.

The content panel height can be specified as needed, and supports internal scrolling if needed. Expanding a content panel does not change the state of any other filter section.

```example
<div className="interstate-docs__usage-demo interstate-sandbox interstate-sandbox">
  <div className=" container" style={{height: 300}}>
    <Accordion>
      <Accordion.Input onChange={e => console.log(e)} title="Makes">
        <Input className='mb-2' type='text' placeholder='Input Accordion' />
      </Accordion.Input>
      <Accordion.Input onChange={e => console.log(e)} title="Models">
        <Input className='mb-2' type='text' placeholder='Input Accordion' />
      </Accordion.Input>
      <Accordion.Input onChange={e => console.log(e)} title="VIN">
        <Input className='mb-2' type='text' placeholder='Input Accordion' />
      </Accordion.Input>
    </Accordion>
  </div>
</div>
```

### Menu Accordion

Each menu accordion content panel accommodates the height of its content and does not allow internal scrolling. Expanding one content panel collapses any other expanded section.

```example
<div className="interstate-docs__usage-demo interstate-sandbox interstate-sandbox">
  <div className=" container" style={{height: 350 }}>
    <Accordion collapseSiblings={false}>
      <Accordion.Item title="Item 1">
        <span>Accordion Content</span>
      </Accordion.Item>
      <Accordion.Item title="Item 2">
        <span>Accordion Content</span>
      </Accordion.Item>
      <Accordion.Item title="Item 3">
        <span>Accordion Content</span>
      </Accordion.Item>
    </Accordion>
  </div>
</div>
```

### Accordion Closed

```jsx dead
<Accordion open={false}>
  <Accordion.Item title="Item 1">
    <p>Accordion Content</p>
  </Accordion.Item>
  <Accordion.Item title="Item 2">
    <p>Accordion Content</p>
  </Accordion.Item>
</Accordion>
```

_Accordion Items are all open by default unless open is set to false._

---

### Open an Accordion Item

```jsx dead
<Accordion open={false}>
  <Accordion.Item defaultOpen={true} title="Item 1">
    <p>Accordion Content</p>
  </Accordion.Item>
  <Accordion.Item title="Item 2">
    <p>Accordion Content</p>
  </Accordion.Item>
</Accordion>
```

_To set an Accordion Item to open by default, set defaultOpen to true on that item._

### Clearble Accordion Input

```jsx dead
<Accordion>
  <Accordion.Input
    onChange={(e) => console.log("example", e)}
    onClear={(clearState) => console.log(clearState)}
    variant="filter"
    title="Makes"
  >
    <Searchable>
      <Searchable.Search />
      <Searchable.List
        data={[
          { label: "Audi", count: 32, value: "Audi", id: "_audi_" },
          { label: "BMW", count: 12, value: "BMW", id: "_bmw_" },
        ]}
      />
    </Searchable>
  </Accordion.Input>
</Accordion>
```

_The Accordion.Input is a controlled component, since it is meant to be used with inputs as children. To use the clearble/onClear feature, manage the state of the component from your app. The onClear function returns a boolean value: true when it is clicked. _
